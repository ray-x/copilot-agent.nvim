1. INTRODUCTION                                   *copilot-agent-intro*

copilot-agent.nvim bridges the official GitHub Copilot SDK to Neovim via a
lightweight Go HTTP service. It provides:

  • Full agentic tool execution (file read/write, terminal, web search)
  • Persistent sessions scoped per working directory
  • Streamed assistant responses with incremental rendering
  • LSP code actions (explain, fix, add tests, add docs)
  • Granular permission management
  • Model switching mid-session with tab-completion
  • File, folder, image, and clipboard image attachments
  • Fuzzy picker integration (snacks, telescope, fzf-lua, mini.pick)
  • Slash commands and @-file completions

Architecture overview:

  Neovim (Lua plugin) ──curl──► Go HTTP service ──SDK──► Copilot CLI/API
                       ◄──SSE──                  ◄──events──

The Go binary runs a single process serving both the HTTP bridge and an LSP
server on stdio. Neovim starts it as an LSP client; all chat traffic uses
curl shell-outs for HTTP and SSE.

2. REQUIREMENTS                                   *copilot-agent-requirements*

  • Neovim 0.11+ (0.12+ recommended)
  • `curl` on PATH
  • Go 1.24+ (only when building from source)
  • GitHub Copilot CLI runtime (`@github/copilot/index.js`)
    or an already-running Copilot CLI server (`-cli-url`)
  • A valid GitHub Copilot subscription

Optional:
  • pngpaste (macOS)      — clipboard image paste
  • wl-paste (Linux/Wayland) — clipboard image paste
  • xclip or xsel (Linux/X11) — clipboard image paste
  • delta                  — rich diff viewer for permission "Show diff"
  • snacks.picker, telescope, fzf-lua, or mini.pick — fuzzy file picker

Run |:checkhealth| copilot_agent to verify all requirements.

3. INSTALLATION                                   *copilot-agent-installation*

lazy.nvim: >lua
  {
    "ray-x/copilot-agent.nvim",
    config = function()
      require("copilot_agent").setup({
        service = { auto_start = true },
      })
    end,
  }
<

Step 1 — Download the pre-built binary (no Go required): >
  :CopilotAgentInstall
<
This downloads the correct binary for your platform and saves it to
`<plugin_root>/bin/copilot-agent` (or `.exe` on Windows). The binary is
used automatically next time the service starts.

Supported platforms:
  linux-amd64        Linux x86_64
  linux-arm64        Linux aarch64
  windows-amd64.exe  Windows x86_64
  windows-arm64.exe  Windows aarch64
  darwin-arm64       macOS Apple Silicon

Alternatively, build from source (requires Go 1.24+): >bash
  cd server/
  go build -o copilot-agent .
<
Or point `service.command` at any existing binary: >lua
  service = { auto_start = true, command = { "/path/to/copilot-agent" } }
<

4. SETUP                                          *copilot-agent-setup*

Call `require("copilot_agent").setup(opts)` with any overrides. All keys are
optional — defaults are shown below: >lua

  require("copilot_agent").setup({
    base_url         = "http://127.0.0.1:8088",  -- for external services
    client_name      = "nvim-copilot",
    permission_mode  = "approve-all",
    auto_create_session = true,
    notify           = true,
    file_log_level   = "WARN",  -- DEBUG logs HTTP/SSE payloads, plugin actions, and transform decisions to stdpath("log") .. "/copilot_agent.log"

    session = {
      working_directory        = nil,   -- function() or string; nil = cwd
      model                    = nil,   -- nil = SDK default
      agent                    = nil,   -- nil = "default"; or "coding", etc.
      streaming                = true,
      enable_config_discovery  = true,  -- respects .github/copilot-instructions.md
      auto_resume              = "prompt",  -- "prompt" (default) | "auto"
    },

    service = {
      auto_start              = false,
      command                 = { "go", "run", "." },
      cwd                     = nil,    -- defaults to <plugin_root>/server
      env                     = nil,
      port_range              = nil,    -- e.g. "18000-19000"
      healthcheck_path        = "/healthz",
      startup_timeout_ms      = 15000,
      startup_poll_interval_ms = 250,
    },

    chat = {
      split                   = "botright vnew",
      title                   = "Copilot Chat",
      system_notify_timeout   = 3000,   -- ms, 0 = never auto-clear
      render_markdown         = false,  -- true = enable render-markdown.nvim
      file_picker             = "auto", -- see |copilot-agent-picker|
    },
  })
<

5. COMMANDS                                       *copilot-agent-commands*

                                                  *:CopilotAgentInstall*
:CopilotAgentInstall
    Download the pre-built `copilot-agent` binary for the current platform
    from the latest GitHub release and save it to `<plugin_root>/bin/`.
    After installation, the binary is used automatically on next startup
    (no Go toolchain required). See |copilot-agent-installation|.

                                                  *:CopilotAgentChat*
:CopilotAgentChat [fullscreen]
    Open the chat buffer. Creates a session if one does not exist.
    With "fullscreen" argument, opens in a new tab instead of a split.

                                                  *:CopilotAgentAsk*
:CopilotAgentAsk [prompt]
    Send [prompt] to the active session. If no argument is given, opens
    `vim.ui.input()` for interactive entry.

                                                  *:CopilotAgentNewSession*
:CopilotAgentNewSession
    Disconnect the current session and create a fresh one in the same
    working directory.

                                                  *:CopilotAgentSwitchSession*
:CopilotAgentSwitchSession
    Show a picker of all persisted sessions (across all projects) and
    switch to the selected one. Sessions are listed newest-first with
    their auto-generated summary. The active session is marked with ●.
    A "New session" option is available at the bottom of the list.
    Rename the current session with the `/rename` slash command.

                                                  *:CopilotAgentDeleteSession*
:CopilotAgentDeleteSession
    Show a newest-first picker of persisted sessions and delete the
    selected one. The picker includes each session's exact ID to make
    destructive selections unambiguous. The active session is marked
    with ●.

                                                  *:CopilotAgentModel*
:CopilotAgentModel [id]
    Switch to model [id]. Without argument, opens an interactive picker.
    Supports tab-completion from the list returned by the service.

                                                  *:CopilotAgentStart*
:CopilotAgentStart
    Manually start the Go service (useful when `auto_start = false`).

                                                  *:CopilotAgentStop*
:CopilotAgentStop[!]
    Disconnect the active session. With [!] also deletes persisted session
    state so the next open starts completely fresh. Deleted sessions keep
    their checkpoint worktree in a soft-deleted state for 7 days before
    automatic cleanup.

                                                  *:CopilotAgentStatus*
:CopilotAgentStatus
    Print service URL, session id, model, and stream status to a float.

                                                  *:CopilotAgentLsp*
:CopilotAgentLsp
    Start (or reuse) the Copilot LSP client for code actions.
    See |copilot-agent-lsp|.

                                                  *:CopilotAgentPasteImage*
:CopilotAgentPasteImage
    Save the current clipboard image to a temp file and add it as a
    pending attachment. Equivalent to <M-v> in the input buffer.
    See |copilot-agent-clipboard|.

                                                  *:CopilotAgentCancel*
:CopilotAgentCancel
    Cancel the current agent turn. Also mapped to <C-c> in the chat
    buffer.

6. INPUT BUFFER                                   *copilot-agent-input*

Open the input buffer from the chat window by pressing `i` or `<Enter>`.
Type your prompt and press `<CR>` or `<C-s>` to send.

------------------------------------------------------------------------------
6.1 Keybindings                                   *copilot-agent-keybindings*

Normal and insert mode (unless noted):

  <CR> / <C-s>        Send the prompt
  q / <Esc>           Close the input buffer (normal mode only)
  <C-t>               Cycle chat mode: ask → plan → agent → autopilot
  <M-a>               Cycle permission mode (see |copilot-agent-permissions|)
  <M-m>               Open the model picker (see |copilot-agent-models|)
  <C-a>               Open the attachment picker menu
  <M-v>               Paste image from clipboard as attachment
  <C-x>               Toggle individual session tools on/off
  <Tab>               Trigger completion (@file or /slash-command)
  <C-p> / <M-p>       Previous prompt from history
  <C-n> / <M-n>       Next prompt from history
  ? (normal)          Show help with keybindings and session commands

------------------------------------------------------------------------------
6.2 Attachments                                   *copilot-agent-attachments*

Press <C-a> to open the attachment picker. Options:

  Current buffer        Attach the previously-focused file buffer
  Visual selection      Attach the last visual selection (file + line range)
  File                  Pick one or more files via fuzzy finder
  Folder                Pick a directory
  Instructions file     Pick a file to use as instructions context (📋)
  Image file            Pick an image file (png, jpg, gif, …) (🖼️)
  Paste image from clipboard  Save clipboard PNG and attach it

Pending attachments are shown in the statusline as `📎 N`. After sending,
each attachment is listed below the prompt text in the chat buffer.

                                                  *copilot-agent-picker*
Fuzzy Picker Integration ~

File / Folder / Image / Instructions choices use the best available picker:

  Priority  Picker           Notes
  --------  ------           -----
  1         snacks.picker    files / directories
  2         telescope        find_files; telescope-file-browser for dirs
  3         fzf-lua          fzf.files; fzf_exec fd --type d for dirs
  4         mini.pick        builtin.files (files only)
  5         vim.ui.input     fallback — type path with completion

Override with `chat.file_picker` in |copilot-agent-setup|: >lua
  chat = { file_picker = "telescope" }  -- force telescope
  chat = { file_picker = "native" }     -- always use vim.ui.input
<

                                                  *copilot-agent-clipboard*
Clipboard Image Paste ~

<M-v> or |:CopilotAgentPasteImage| saves the clipboard image to a temporary
PNG file and adds it as an attachment. The temp file is deleted automatically
after the message is sent (or when the session ends / Neovim exits).

Required tools by platform:

  Platform        Tool        Install
  --------        ----        -------
  macOS           pngpaste    brew install pngpaste
  Linux/Wayland   wl-paste    sudo apt install wl-clipboard
  Linux/X11       xclip       sudo apt install xclip
                  xsel        sudo apt install xsel

------------------------------------------------------------------------------
6.3 Chat Modes                                    *copilot-agent-modes*

The current mode is shown in the prompt prefix (e.g. `agent❯`).

  ask        💬  Standard question-and-answer (interactive permission)
  plan       📋  Ask Copilot to create an implementation plan
  agent      🤖  Agentic loop — reads auto-approved, writes prompt (approve-reads)
  autopilot  🚀  Fully autonomous — all tools silently approved (approve-all)

Cycle with <C-t> in the input buffer. Each mode sets a sensible default
permission mode automatically; override with <M-a>.


------------------------------------------------------------------------------
6.4 Slash Commands                                *copilot-agent-slash*

Type `/` in the input buffer and press <Tab> for completions. Examples:

  /clear              Clear the chat transcript
  /copy               Copy the last response to the clipboard
  /help               Show available slash commands
  /newfile            Ask Copilot to create a new file
  /fix                Fix the selected code
  /tests              Generate tests for the selection
  /doc                Generate documentation for the selection

Type `@` followed by a file path for file attachments with autocomplete.

7. PERMISSION MODES                               *copilot-agent-permissions*

Control how tool-use requests are handled. Cycle with <M-a> in the input
buffer, or set via |copilot-agent-setup| or `POST /sessions/{id}/permission-mode`.

  interactive     Neovim shows Allow / Deny for each tool use (🔐)
  approve-reads   Auto-approve workspace reads; prompt for writes/shell (📂)
  approve-all     Auto-approve all tool uses silently (✅) [default]
  autopilot       approve-all + auto-answer ask_user requests (🤖)
  reject-all      Reject all tool uses — safe read-only mode (🚫)

In interactive mode, each permission prompt offers additional options:

  Allow                    Approve this single request
  Show diff                Preview unified diff in a float (write only)
  Deny                     Reject this single request
  Allow this directory     Approve + whitelist the directory (read/write only)
  Allow all for session    Approve + switch to approve-all for the rest

When a write permission includes a diff, "Show diff" pipes it through the
configured `chat.diff_cmd` (default: `delta`) in a floating terminal
buffer. If the command is not found, falls back to a plain diff buffer.
When delta is used and the window is ≥100 columns, `--side-by-side` is
appended automatically. Press `q` or `<Esc>` to close and return to the
Allow/Deny picker.

The current mode is shown by |copilot-agent-statusline| functions.

                                                  *copilot-agent-file-changes*
When the agent creates or modifies files, buffers are automatically
reloaded via `:checktime`. For git-tracked files, a "Review changes?"
prompt offers to open a vimdiff tab showing the HEAD version alongside
the agent's changes. Disable with: >lua
  chat = { diff_review = false }
<

8. MODELS                                         *copilot-agent-models*

Switch model with |:CopilotAgentModel| or <M-m> in the input buffer.
The model list is fetched from the service and cached; tab-completion works
in the command line.

Set a default model in |copilot-agent-setup|: >lua
  session = { model = "claude-sonnet-4.6" }
<

The current model is shown by |copilot-agent-statusline| functions.

9. SESSION MANAGEMENT                             *copilot-agent-sessions*

Sessions are scoped per working directory. When you open the chat, the plugin
looks for a persisted session matching the current directory and resumes it
automatically when there is only one matching session. With multiple sessions
and `auto_resume = "prompt"`, a picker lets you choose which to
resume.

  :CopilotAgentNewSession       Start a fresh session in the same directory
  :CopilotAgentSwitchSession    Pick from all persisted sessions and switch
  :CopilotAgentDeleteSession    Delete a picked session by exact ID
  :CopilotAgentStop             Disconnect (session state persisted on disk)
  :CopilotAgentStop!            Delete active session; checkpoints kept 7 days

When a session is deleted, its checkpoint git worktree is soft-deleted for
7 days before it is pruned automatically on a later startup or checkpoint
operation.

Sessions are auto-named by the SDK after the first conversation turn. You
can rename a session by typing `/rename My Session Name` in the input
buffer.

The active session summary and short session ID are shown in the
chat/output statusline. Transcript separators between turns also render the
completed-turn checkpoint label (for example `v001`) as a virtual rule so it is easy to copy for rewind or recovery
workflows. See |copilot-agent-statusline|.

Multiple Neovim instances can connect to separate sessions on the same
service by using `service.port_range`: >lua
  service = { port_range = "18000-19000" }
<
Each instance binds to the first free port in the range.

10. LSP CODE ACTIONS                              *copilot-agent-lsp*

The Go binary runs an LSP server on stdio. Start it with |:CopilotAgentLsp|
or `require("copilot_agent").start_lsp()`.

Available code actions (trigger with `vim.lsp.buf.code_action()` on any
selection):

  Explain selection     Ask Copilot to explain the selected code
  Fix selection         Ask Copilot to suggest a fix
  Add tests             Generate unit tests for the selection
  Add docs              Generate documentation for the selection

The action builds a prompt from the selection, file path, and line range,
then posts it to the active HTTP session.

Call `start_lsp()` in setup to make code actions available immediately: >lua
  require("copilot_agent").setup({ ... })
  require("copilot_agent").start_lsp()
<

11. STATUSLINE API                                *copilot-agent-statusline*

Each function returns a short display string and is safe to call at any time.

  require("copilot_agent").statusline_mode()        -- [ask] / [plan] / [agent] / [autopilot]
  require("copilot_agent").statusline_model()       -- claude-sonnet-4.6
  require("copilot_agent").statusline_busy()        -- ✅ready / ⏳working / 📝sync / 🧩2 tasks / ❓input
  require("copilot_agent").statusline_permission()  -- 🔐interactive / ✅approve-all / …
  require("copilot_agent").statusline_attachments() -- 📎3 (only when N > 0)
  require("copilot_agent").statusline_tool()        -- 🔧 read_file (active tool)
  require("copilot_agent").statusline_intent()      -- current agent intent
  require("copilot_agent").statusline_context()     -- 12k/200k (token usage)

lualine example: >lua
  require("lualine").setup({
    sections = {
      lualine_x = {
        require("copilot_agent").statusline_mode,
        require("copilot_agent").statusline_model,
        require("copilot_agent").statusline_busy,
        require("copilot_agent").statusline_permission,
        require("copilot_agent").statusline_attachments,
      }
    }
  })
<

heirline / &statusline: >vim
  %{v:lua.require'copilot_agent'.statusline()}
<

12. CONFIGURATION                                 *copilot-agent-config*

All configuration keys and their defaults:

base_url ~
  Type: string
  Default: `"http://127.0.0.1:8088"`
  Base URL for connecting to an externally-started service. Overwritten
  automatically when `service.auto_start = true`.

client_name ~
  Type: string
  Default: `"nvim-copilot"`
  Name sent to the Copilot SDK when creating sessions.

permission_mode ~
  Type: string  ("interactive" | "approve-all" | "autopilot" | "reject-all")
  Default: `"approve-all"`
  Initial permission mode for new sessions. See |copilot-agent-permissions|.

auto_create_session ~
  Type: boolean
  Default: `true`
  When true, eagerly create a session in the background on startup.

notify ~
  Type: boolean
  Default: `true`
  Set false to suppress all `[copilot-agent]` vim.notify calls.

session.working_directory ~
  Type: string | function() → string | nil
  Default: `nil` (uses vim.fn.getcwd())
  Working directory for sessions. Accepts a function for dynamic values.

session.model ~
  Type: string | nil
  Default: `nil` (SDK picks a default)
  Default model for new sessions.

session.agent ~
  Type: string | nil
  Default: `nil` ("default" agent)
  Agent to use. Examples: "coding", "gpt-4.1", or a custom agent name.

session.streaming ~
  Type: boolean
  Default: `true`
  Stream assistant responses token by token.

session.enable_config_discovery ~
  Type: boolean
  Default: `true`
  Respect `.github/copilot-instructions.md` and similar project files.

session.auto_resume ~
  Type: string  ("prompt" | "auto")
  Default: `"prompt"`
  How to handle multiple matching sessions on startup.
  "prompt" — show a picker (most recent first) so you can choose which session
              to resume or start a new one.  This is the default.
  "auto"   — silently resume the most recent session without prompting.

service.auto_start ~
  Type: boolean
  Default: `false`
  Start the Go service automatically when the plugin is loaded.

service.command ~
  Type: string[] | function() → string[] | nil
  Default: `nil`
  Command used to start the service. `nil` (default) = auto-detect:
  uses `<plugin_root>/bin/copilot-agent` if present (installed via
  |:CopilotAgentInstall|), otherwise falls back to `{ "go", "run", "." }`.
  Set explicitly to override: `{ "/path/to/copilot-agent" }`.

service.cwd ~
  Type: string | nil
  Default: `nil` (uses `<plugin_root>/server`)
  Working directory for the service process.

service.env ~
  Type: table | nil
  Default: `nil`
  Extra environment variables for the service process.

service.port_range ~
  Type: string | nil  (format: "lo-hi", e.g. "18000-19000")
  Default: `nil` (OS picks a free port)
  Restrict the service to a fixed port range.

service.startup_timeout_ms ~
  Type: integer
  Default: `15000`
  Milliseconds to wait for the service to become healthy.

service.startup_poll_interval_ms ~
  Type: integer
  Default: `250`
  Interval between health-check polls during startup.

chat.split ~
  Type: string
  Default: `"botright vnew"`
  Vim split command used to open the chat window.

chat.title ~
  Type: string
  Default: `"Copilot Chat"`
  Title line displayed at the top of the chat buffer.

chat.system_notify_timeout ~
  Type: integer
  Default: `3000`
  Milliseconds before transient system notices are auto-cleared from the
  cmdline area. Set to 0 to never auto-clear.

chat.render_markdown ~
  Type: boolean
  Default: `false`
  Enable render-markdown.nvim on the chat buffer. When true, the plugin
  calls rm.refresh() once per completed turn. Disable if you see visual
  lag on long responses.

chat.file_picker ~
  Type: string  ("auto"|"snacks"|"telescope"|"fzf-lua"|"mini.pick"|"native")
  Default: `"auto"`
  Fuzzy picker to use for file/folder/image attachment.
  See |copilot-agent-picker|.

chat.diff_review ~
  Type: boolean
  Default: `true`
  When true, offer a vimdiff review after the agent modifies a git-tracked
  file. Set false to just notify and auto-reload the buffer.

chat.diff_cmd ~
  Type: string[] | false
  Default: `{ "delta" }`
  External diff command for the permission "Show diff" viewer. The diff
  text is piped via stdin. Examples: `{ "delta" }`,
  `{ "delta", "--side-by-side" }`, `{ "diff-so-fancy" }`.
  When set to `{ "delta" }` and the window is ≥100 columns,
  `--side-by-side` is appended automatically.
  Falls back to a builtin floating diff buffer if the command is not found.
  Set to `false` to always use the builtin diff viewer.

chat.fullscreen ~
  Type: boolean
  Default: `false`
  When true, `:CopilotAgentChat` opens in a new tab by default.
  Can also be triggered per-invocation: `:CopilotAgentChat fullscreen`.

13. HTTP API                                      *copilot-agent-api*

The Go service exposes a REST + SSE API on the bound address.

Endpoints: ~

  GET  /healthz                            Health check
  GET  /sessions                           List all sessions
  POST /sessions                           Create or resume a session
  GET  /sessions/{id}                      Get session metadata
  DELETE /sessions/{id}                    Disconnect; ?delete=true removes state
  GET  /sessions/{id}/messages             Fetch stored session events
  POST /sessions/{id}/messages             Send a prompt with attachments
  GET  /sessions/{id}/events               SSE stream of events
  POST /sessions/{id}/user-input/{reqID}   Answer a pending ask_user request
  POST /sessions/{id}/permission/{reqID}   Answer a permission request
  POST /sessions/{id}/permission-mode      Update permission mode
  POST /sessions/{id}/model               Switch model
  POST /sessions/{id}/tools               Update excluded tools list
  GET  /models                             List available models

POST /sessions request body: >json
  {
    "workingDirectory": "/path/to/project",
    "permissionMode":   "approve-all",
    "clientName":       "my-client",
    "model":            "claude-sonnet-4.6",
    "streaming":        true
  }
<

POST /sessions/{id}/messages request body: >json
  {
    "prompt": "Explain this code",
    "mode": "agent",
    "attachments": [
      { "type": "file", "path": "/abs/path/to/file.go" },
      { "type": "selection", "filePath": "/abs/path/to/file.go",
        "text": "selected text", "lineRange": {"start": 0, "end": 5} }
    ]
  }
<

SSE Event Types: ~

  session.event              Raw SDK events (assistant.message_delta, etc.)
  host.user_input_requested  Agent needs user input
  host.permission_requested  Tool use needs approval (interactive mode)
  host.permission_decision   Approval/rejection outcome
  host.permission_mode_changed  Permission mode updated
  host.model_changed         Model switched
  host.session_disconnected  Session closed
  : keepalive                SSE keepalive comment (ignore)

Service flags: ~

  -addr          HTTP listen address (default: OS picks a free port)
  -port-range    Try ports lo-hi, first free wins (e.g. 18000-19000)
  -cwd           Default working directory for sessions
  -model         Default model for new sessions
  -cli-path      Path to Copilot CLI binary / index.js
  -cli-url       URL of an already-running Copilot CLI server
  -log-level     Copilot CLI log level
  -lsp           Start LSP server on stdio (default: true)

14. LUA API                                       *copilot-agent-lua*

All functions are members of `require("copilot_agent")`.

setup({opts})                             *copilot-agent.setup()*
  Initialize the plugin. See |copilot-agent-setup|. Returns the module.

open_chat()                               *copilot-agent.open_chat()*
  Ensure the chat window is open. Creates a session if needed.
  Returns the chat buffer number.

new_session()                             *copilot-agent.new_session()*
  Disconnect the current session and start a fresh one.

switch_session()                          *copilot-agent.switch_session()*
  Show a picker of all persisted sessions and switch to the selected one.
  See |:CopilotAgentSwitchSession|.

delete_session()                          *copilot-agent.delete_session()*
  Show a picker of all persisted sessions and delete the selected one.
  Deleted sessions leave their checkpoint repo in a soft-deleted state
  for 7 days before it is pruned automatically.
